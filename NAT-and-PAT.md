# NAT and PAT

## كيف تخرج الأجهزة الداخلية إلى الإنترنت؟

في البيت أو الشركة، أغلب الأجهزة تستخدم IP خاص مثل:

```text
192.168.1.10
192.168.1.20
10.0.0.5
```

هذه العناوين تسمى:

```text
Private IP Addresses
```

وهي لا تظهر مباشرة على الإنترنت.

لكن عندما تفتح موقعًا مثل:

```text
youtube.com
google.com
facebook.com
```

يجب أن يظهر طلبك على الإنترنت بعنوان عام يسمى:

```text
Public IP Address
```

هنا يظهر دور:

```text
NAT
```

---

# ما هو NAT؟

NAT اختصار لـ:

```text
Network Address Translation
```

يعني:

```text
ترجمة عناوين الشبكة
```

ببساطة، NAT يجعل الأجهزة التي تستخدم Private IP تستطيع الخروج إلى الإنترنت باستخدام Public IP.

مثال:

```text
Before NAT:
Source IP = 192.168.1.10

After NAT:
Source IP = 41.33.20.5
```

يعني الراوتر غيّر عنوان الجهاز الداخلي إلى عنوان عام يمكن استخدامه على الإنترنت.

---

# لماذا نحتاج NAT؟

نحتاج NAT لسببين مهمين:

```text
1. Private IP لا يعمل مباشرة على الإنترنت
2. عدد IPv4 محدود
```

لو كل جهاز في العالم احتاج Public IP خاص به، لن تكفي عناوين IPv4.

لذلك نستخدم Private IP داخل الشبكات المحلية، ونستخدم NAT عند الخروج للإنترنت.

---

# Private IP vs Public IP

| المقارنة | Private IP | Public IP |
|---|---|---|
| المكان | داخل الشبكة المحلية | على الإنترنت |
| مثال | 192.168.1.10 | 41.33.20.5 |
| فريد عالميًا؟ | لا | نعم |
| يخرج للإنترنت مباشرة؟ | لا | نعم |
| يحتاج NAT؟ | نعم | لا غالبًا |

---

# نطاقات Private IP

هذه أشهر نطاقات الـ Private IP:

```text
10.0.0.0      - 10.255.255.255
172.16.0.0    - 172.31.255.255
192.168.0.0   - 192.168.255.255
```

مثال في البيت:

```text
Router = 192.168.1.1
Laptop = 192.168.1.10
Phone  = 192.168.1.11
TV     = 192.168.1.12
```

كل هذه الأجهزة تخرج للإنترنت غالبًا من خلال Public IP واحد.

---

# تشبيه بسيط

تخيل شركة كبيرة لها عنوان واحد على الشارع:

```text
عنوان الشركة العام = Public IP
```

لكن داخل الشركة توجد مكاتب كثيرة:

```text
مكتب 101
مكتب 102
مكتب 103
```

هذه مثل:

```text
Private IPs
```

عندما يرسل موظف رسالة للخارج، موظف الاستقبال يضع عنوان الشركة العام على الرسالة.

وعندما يأتي الرد، موظف الاستقبال يعرف لأي مكتب يوصله.

في الشبكات:

```text
موظف الاستقبال = Router
عنوان الشركة العام = Public IP
رقم المكتب = Private IP
جدول الاستقبال = NAT Table
```

---

# أين يعمل NAT؟

NAT يعمل غالبًا على الراوتر أو الفايروول.

مثال:

```text
Home Router
Company Firewall
Linux Gateway
Cloud NAT Gateway
```

NAT يغير معلومات داخل الـ Packet، خصوصًا:

```text
Source IP
Source Port أحيانًا
Destination IP في Port Forwarding
Destination Port أحيانًا
```

---

# أنواع NAT

هناك 3 أنواع مهمة:

```text
Static NAT
Dynamic NAT
PAT / NAT Overload
```

---

# 1. Static NAT

Static NAT يعني ترجمة ثابتة واحد لواحد.

مثال:

```text
192.168.1.10  ↔  203.0.113.10
192.168.1.11  ↔  203.0.113.11
```

كل Private IP له Public IP ثابت.

---

## متى نستخدم Static NAT؟

نستخدمه عندما يكون عندنا جهاز داخلي نريد أن يكون له عنوان عام ثابت.

مثال:

```text
Web Server
Mail Server
Camera Server
VPN Server
```

مثال:

```text
Internal Server = 192.168.1.10
Public IP       = 203.0.113.10
```

أي شخص من الإنترنت يتصل بـ:

```text
203.0.113.10
```

يتم تحويله إلى:

```text
192.168.1.10
```

---

## عيب Static NAT

العيب أنه يحتاج Public IP لكل جهاز.

وهذا مكلف وغير عملي في أغلب الشبكات.

---

# 2. Dynamic NAT

Dynamic NAT يعني أن الراوتر لديه مجموعة Public IPs تسمى:

```text
Pool
```

ويعطي الأجهزة الداخلية عنوانًا عامًا من هذه المجموعة عند الحاجة.

مثال:

```text
Private IPs:
192.168.1.10
192.168.1.11
192.168.1.12

Public Pool:
203.0.113.10
203.0.113.11
203.0.113.12
```

كل جهاز يخرج يأخذ Public IP متاحًا من الـ Pool.

---

## عيب Dynamic NAT

لو عدد الأجهزة أكبر من عدد Public IPs، بعض الأجهزة لن تستطيع الخروج حتى يصبح هناك Public IP فارغ.

مثال:

```text
Public Pool = 3 IPs
Devices     = 10 Devices
```

إذن فقط 3 أجهزة يمكنها الخروج في نفس الوقت.

---

# 3. PAT / NAT Overload

PAT اختصار لـ:

```text
Port Address Translation
```

ويسمى أيضًا:

```text
NAT Overload
```

وهذا هو النوع الأشهر في البيوت والشركات.

الفكرة:

```text
أجهزة كثيرة تستخدم Public IP واحد
لكن ببورتات مختلفة
```

مثال:

```text
192.168.1.10:50001 → 41.33.20.5:60001
192.168.1.11:50002 → 41.33.20.5:60002
192.168.1.12:50003 → 41.33.20.5:60003
```

كلهم خرجوا بنفس Public IP:

```text
41.33.20.5
```

لكن الراوتر يميز بينهم باستخدام Port Number.

---

# لماذا PAT مهم جدًا؟

لأن البيت كله يمكنه استخدام Public IP واحد فقط.

مثال:

```text
Laptop
Phone
Tablet
Smart TV
PlayStation
Printer
```

كل هذه الأجهزة قد تخرج إلى الإنترنت من نفس Public IP، والراوتر يفرق بينها بالبورتات.

---

# مثال عملي على PAT

جهازك يريد فتح YouTube.

قبل NAT:

```text
Source IP      = 192.168.1.10
Source Port    = 50001
Destination IP = 142.250.80.46
Destination Port = 443
Protocol       = TCP
```

بعد NAT/PAT:

```text
Source IP      = 41.33.20.5
Source Port    = 62001
Destination IP = 142.250.80.46
Destination Port = 443
Protocol       = TCP
```

الراوتر غيّر:

```text
Source IP
Source Port
```

ثم سجل هذا في NAT Table.

---

# NAT Table

NAT Table هو جدول داخل الراوتر يتذكر التحويلات.

مثال:

```text
Inside Local        Inside Global       Outside Global
192.168.1.10:50001  41.33.20.5:62001   142.250.80.46:443
192.168.1.11:50002  41.33.20.5:62002   157.240.1.35:443
```

معنى الجدول:

```text
لو جاء رد إلى 41.33.20.5:62001
أرسله إلى 192.168.1.10:50001
```

```text
لو جاء رد إلى 41.33.20.5:62002
أرسله إلى 192.168.1.11:50002
```

---

# رحلة الطلب مع NAT

## 1. الجهاز يرسل الطلب

```text
192.168.1.10:50001 → 142.250.80.46:443
```

الجهاز يرسل الطلب إلى الراوتر لأنه خارج الشبكة المحلية.

---

## 2. الراوتر ينفذ NAT/PAT

الراوتر يغير المصدر:

```text
192.168.1.10:50001
```

إلى:

```text
41.33.20.5:62001
```

ثم يرسل الطلب إلى الإنترنت.

---

## 3. السيرفر يرد

السيرفر يرى الطلب قادمًا من:

```text
41.33.20.5:62001
```

فيرسل الرد إلى نفس العنوان والبورت.

---

## 4. الراوتر يستقبل الرد

الراوتر يرى الرد:

```text
Destination = 41.33.20.5:62001
```

يفتح NAT Table ويجد:

```text
41.33.20.5:62001 → 192.168.1.10:50001
```

---

## 5. الراوتر يعيد الرد للجهاز الداخلي

الراوتر يغير الوجهة ويرسل الرد إلى:

```text
192.168.1.10:50001
```

وهكذا تصل البيانات للجهاز الصحيح.

---

# رسم مبسط

```text
┌──────────────┐        ┌──────────────┐        ┌──────────────┐
│   Laptop     │        │    Router    │        │   Internet   │
│192.168.1.10  │        │41.33.20.5    │        │142.250.80.46 │
└──────┬───────┘        └──────┬───────┘        └──────┬───────┘
       │                       │                       │
       │ 192.168.1.10:50001    │                       │
       │──────────────────────>│                       │
       │                       │ 41.33.20.5:62001      │
       │                       │──────────────────────>│
       │                       │                       │
       │                       │<──────────────────────│
       │                       │ Reply to 41.33.20.5   │
       │<──────────────────────│                       │
       │ Reply to 192.168.1.10 │                       │
```

---

# Inside Local و Inside Global

في NAT، توجد مصطلحات مهمة.

## Inside Local

هو عنوان الجهاز داخل الشبكة المحلية.

مثال:

```text
192.168.1.10
```

---

## Inside Global

هو العنوان الذي يظهر به الجهاز على الإنترنت بعد NAT.

مثال:

```text
41.33.20.5
```

---

## Outside Global

هو عنوان السيرفر الخارجي الحقيقي.

مثال:

```text
142.250.80.46
```

---

## Outside Local

في أغلب الحالات البسيطة يكون نفس Outside Global، ولا تحتاج التركيز عليه في البداية.

---

# مقارنة NAT و PAT

| المقارنة | NAT | PAT |
|---|---|---|
| يغير | IP Address | IP Address + Port |
| عدد الأجهزة | حسب عدد Public IPs | أجهزة كثيرة على Public IP واحد |
| الاستخدام | أقل في البيوت | الأكثر استخدامًا |
| مثال | 192.168.1.10 → 203.0.113.10 | 192.168.1.10:50001 → 203.0.113.10:62001 |

---

# هل NAT يعمل مع TCP فقط؟

لا.

NAT/PAT يمكن أن يعمل مع:

```text
TCP
UDP
ICMP
```

لكن PAT يعتمد كثيرًا على البورتات، والبورتات موجودة في TCP و UDP.

أما ICMP لا يستخدم Ports، لذلك الراوتر يستخدم معلومات أخرى مثل:

```text
ICMP Identifier
```

حتى يميز الطلبات.

---

# NAT و TCP

عند فتح موقع HTTPS:

```text
Destination Port = 443
Protocol = TCP
```

الراوتر ينشئ NAT Entry ويربط الاتصال بالبورتات.

مثال:

```text
192.168.1.10:51000 → 41.33.20.5:62001 → 142.250.80.46:443
```

---

# NAT و UDP

UDP لا يملك Connection مثل TCP، لكن الراوتر ينشئ NAT Entry مؤقت.

مثال DNS:

```text
192.168.1.10:53000 → 8.8.8.8:53
```

بعد NAT:

```text
41.33.20.5:62010 → 8.8.8.8:53
```

لو لم يرجع رد خلال وقت معين، يحذف الراوتر الـ Entry.

---

# NAT و ICMP Ping

عندما تعمل Ping من جهاز داخلي إلى 8.8.8.8:

```text
ping 8.8.8.8
```

الراوتر يترجم Source IP.

ويستخدم معلومات ICMP حتى يعرف الرد راجع لأي جهاز.

---

# NAT و Firewall

مهم جدًا:

```text
NAT ليس Firewall
```

NAT يترجم العناوين.

Firewall يسمح أو يمنع الترافيك.

لكن في راوتر البيت، الوظيفتان غالبًا موجودتان في نفس الجهاز.

مثال:

```text
NAT = يغير 192.168.1.10 إلى Public IP
Firewall = يمنع اتصال خارجي غير مسموح
```

---

# لماذا الاتصالات القادمة من الإنترنت لا تدخل بسهولة؟

في PAT العادي، الراوتر يسمح غالبًا بالردود على اتصالات بدأت من الداخل.

مثال:

```text
Laptop بدأ اتصال مع YouTube
YouTube يرد
الراوتر يسمح بالرد لأنه موجود في NAT Table
```

لكن لو شخص من الخارج حاول يبدأ اتصال بجهازك الداخلي:

```text
Internet → 41.33.20.5:3389
```

الراوتر لا يعرف لأي جهاز داخلي يرسل هذا الطلب، إلا إذا كان هناك إعداد مثل:

```text
Port Forwarding
```

---

# Port Forwarding

Port Forwarding يعني توجيه بورت معين من Public IP إلى جهاز داخلي.

مثال:

```text
Public IP:3389 → 192.168.1.10:3389
```

أو:

```text
Public IP:80 → 192.168.1.20:80
```

هذا يستخدم عندما تريد تشغيل خدمة داخل الشبكة والوصول لها من الخارج.

أمثلة:

```text
Web Server
Game Server
Remote Desktop
Camera System
VPN Server
```

---

# مثال Port Forwarding

عندك Web Server داخلي:

```text
Server IP = 192.168.1.20
Service   = HTTP
Port      = 80
```

تعمل Port Forwarding:

```text
41.33.20.5:80 → 192.168.1.20:80
```

أي شخص يفتح:

```text
http://41.33.20.5
```

الراوتر يرسل الطلب إلى السيرفر الداخلي.

---

# Port Forwarding وخطورة الأمان

Port Forwarding يفتح بابًا من الإنترنت إلى جهاز داخلي.

لذلك يجب الحذر.

نصائح:

```text
لا تفتح بورتات بدون حاجة
استخدم كلمات مرور قوية
حدّث الجهاز أو السيرفر
استخدم VPN بدل فتح Remote Desktop مباشرة
اسمح لعناوين IP معينة فقط إن أمكن
راقب Logs
```

---

# DMZ

في بعض الراوترات يوجد خيار اسمه:

```text
DMZ
```

غالبًا في راوترات البيت، DMZ يعني إرسال معظم الاتصالات القادمة من الإنترنت إلى جهاز داخلي معين.

مثال:

```text
All incoming traffic → 192.168.1.50
```

هذا خطير إذا لم تكن فاهمًا ماذا تفعل، لأنه قد يعرض الجهاز مباشرة لهجمات الإنترنت.

---

# CGNAT

CGNAT اختصار لـ:

```text
Carrier-Grade NAT
```

ويعني أن مزود خدمة الإنترنت نفسه يستخدم NAT.

في هذه الحالة، الراوتر في بيتك قد لا يحصل على Public IP حقيقي، بل يحصل على IP خاص أو شبه خاص من مزود الخدمة.

مثال WAN IP على الراوتر:

```text
100.64.x.x
10.x.x.x
172.16.x.x
192.168.x.x
```

لو أنت خلف CGNAT، قد لا يعمل Port Forwarding من الخارج بسهولة.

---

# كيف تعرف أنك خلف CGNAT؟

قارن بين:

```text
WAN IP داخل صفحة الراوتر
Public IP الظاهر على مواقع معرفة IP
```

لو مختلفين بشكل واضح، قد تكون خلف CGNAT.

مثال:

```text
Router WAN IP = 100.72.10.5
What is my IP = 41.33.20.5
```

هذا غالبًا يعني CGNAT.

---

# حل مشكلة CGNAT

الحلول المحتملة:

```text
طلب Public IP حقيقي من مزود الخدمة
استخدام VPN بخصائص Port Forwarding
استخدام Cloud Tunnel
استخدام VPS كوسيط
استخدام IPv6 إذا متاح
```

---

# NAT و IPv6

IPv6 صمم أساسًا ليعطي عددًا ضخمًا جدًا من العناوين.

لذلك الفكرة الأصلية في IPv6 أن كل جهاز يمكن أن يحصل على عنوان عالمي بدون NAT.

لكن هذا لا يعني أن الجهاز مكشوف بدون حماية.

في IPv6 نحتاج Firewall جيد بدل الاعتماد على NAT كنوع من العزل غير المباشر.

---

# هل NAT يزيد الأمان؟

NAT قد يمنع الاتصالات القادمة العشوائية لأنه لا يوجد NAT Entry لها.

لكن لا يجب اعتباره حماية كاملة.

الأمان الحقيقي يحتاج:

```text
Firewall Rules
Updates
Strong Passwords
VPN
Network Segmentation
Monitoring
```

---

# NAT و HTTPS

NAT لا يفك تشفير HTTPS.

عند فتح موقع HTTPS:

```text
NAT يرى IPs و Ports
لكنه لا يستطيع قراءة محتوى HTTPS
```

يعني الراوتر يعرف أنك متصل بـ IP معين على Port 443، لكنه لا يرى محتوى الصفحة إذا الاتصال مشفر بشكل صحيح.

---

# NAT في رحلة فتح موقع

أنت فتحت:

```text
https://google.com
```

الخطوات:

```text
1. DHCP أعطى جهازك IP و Gateway و DNS
2. DNS يحول google.com إلى IP
3. الجهاز يعرف أن Google خارج الشبكة
4. الجهاز يعمل ARP لمعرفة MAC الراوتر
5. الجهاز يرسل TCP SYN إلى Google على Port 443
6. الراوتر يعمل NAT/PAT
7. Google يرد إلى Public IP الخاص بالراوتر
8. الراوتر يستخدم NAT Table ويرجع الرد لجهازك
9. TLS Handshake يتم بين جهازك وجوجل
10. HTTP Request يرسل داخل TLS
```

---

# ماذا يتغير وماذا لا يتغير؟

| العنصر | هل يتغير مع NAT؟ |
|---|---|
| Source IP | نعم غالبًا |
| Source Port | نعم غالبًا في PAT |
| Destination IP | لا في الاتصال العادي |
| Destination Port | لا غالبًا |
| MAC Address | يتغير عند كل Hop بشكل طبيعي |
| HTTP Data | لا يغيرها NAT عادة |
| TLS Content | لا يستطيع NAT قراءته أو تعديله عادة |

---

# NAT و 5-Tuple

الاتصال يتم تمييزه غالبًا بخمسة عناصر:

```text
Source IP
Source Port
Destination IP
Destination Port
Protocol
```

مثال:

```text
192.168.1.10
50001
142.250.80.46
443
TCP
```

بعد NAT/PAT:

```text
41.33.20.5
62001
142.250.80.46
443
TCP
```

الراوتر يستخدم هذه المعلومات ليعرف أي رد يرجع لأي جهاز.

---

# ماذا يحدث لو جهازان استخدما نفس Source Port؟

مثال:

```text
Laptop = 192.168.1.10:50000
Phone  = 192.168.1.11:50000
```

كلاهما يريد الخروج إلى الإنترنت.

الراوتر يستطيع تغيير البورت الخارجي:

```text
192.168.1.10:50000 → 41.33.20.5:62001
192.168.1.11:50000 → 41.33.20.5:62002
```

وبهذا لا يحدث تعارض.

---

# NAT Timeout

الراوتر لا يحتفظ بـ NAT Entries إلى الأبد.

كل Entry له وقت معين.

لو لم يحدث Traffic لفترة، يتم حذف الـ Entry.

مثال:

```text
TCP Established Timeout
UDP Timeout
ICMP Timeout
```

لو تم حذف الـ Entry، وأتى رد متأخر، قد لا يعرف الراوتر أين يرسله.

# NAT و الألعاب

الألعاب أحيانًا تحتاج اتصال مباشر أو استقبال اتصالات من لاعبين آخرين.

لذلك قد تظهر مشاكل مثل:

```text
Strict NAT
Moderate NAT
Open NAT
```

الحلول قد تشمل:

```text
UPnP
Port Forwarding
Public IP
تجنب CGNAT
```

لكن UPnP قد يكون خطرًا إذا كان يسمح للتطبيقات بفتح بورتات تلقائيًا بدون رقابة.

---

# UPnP

UPnP يسمح للأجهزة أو التطبيقات داخل الشبكة أن تطلب من الراوتر فتح Port Forwarding تلقائيًا.

مثال:

```text
Game Console يطلب فتح Port معين
Router يفتحه تلقائيًا
```

هذا قد يسهل الألعاب وبعض التطبيقات، لكنه قد يكون خطرًا إذا استغله برنامج ضار.

---

# NAT و VPN

عند استخدام VPN، الترافيك يخرج غالبًا داخل Tunnel.

الراوتر المنزلي يعمل NAT للترافيك المتجه إلى VPN Server.

بعد ذلك، VPN Server قد يعمل NAT آخر للترافيك الخارج إلى الإنترنت.

مثال:

```text
Your Device → Home Router NAT → VPN Server → Internet
```

---

# Double NAT

Double NAT يعني وجود NAT مرتين في الطريق.

مثال:

```text
جهازك → راوتر داخلي → راوتر مزود الخدمة → الإنترنت
```

أو:

```text
Router 1 داخل البيت
Router 2 من الشركة أو ISP
```

Double NAT قد يسبب مشاكل في:

```text
Port Forwarding
Online Gaming
VPN
Remote Access
```

---

# حلول Double NAT

```text
جعل أحد الراوترات Bridge Mode
استخدام DMZ من الراوتر الأول إلى الثاني
إلغاء NAT على جهاز واستخدامه Access Point
طلب Public IP من ISP
```

# أوامر مفيدة

## Windows

عرض الاتصالات والبورتات:

```cmd
netstat -n
```

عرض إعدادات الشبكة:

```cmd
ipconfig /all
```

---

## Linux

عرض الاتصالات:

```bash
ss -tunap
```

عرض قواعد NAT في iptables:

```bash
sudo iptables -t nat -L -n -v
```

عرض قواعد NAT في nftables:

```bash
sudo nft list ruleset
```

---

## Cisco

عرض NAT Translations:

```cisco
show ip nat translations
```

عرض إحصائيات NAT:

```cisco
show ip nat statistics
```

مثال Static NAT:

```cisco
ip nat inside source static 192.168.1.10 203.0.113.10
```

مثال PAT:

```cisco
access-list 1 permit 192.168.1.0 0.0.0.255
ip nat inside source list 1 interface g0/1 overload
```

# Troubleshooting NAT

## المشكلة 1: الأجهزة لا تدخل الإنترنت

افحص:

```text
هل الجهاز أخذ IP صحيح؟
هل Default Gateway صحيح؟
هل DNS يعمل؟
هل الراوتر لديه Internet/WAN؟
هل NAT مفعّل؟
هل توجد Default Route؟
```

---

## المشكلة 2: Ping للراوتر يعمل لكن الإنترنت لا يعمل

قد تكون المشكلة في:

```text
NAT
Default Route
ISP
Firewall
DNS
```

---

## المشكلة 3: الإنترنت يعمل لكن Port Forwarding لا يعمل

افحص:

```text
هل الجهاز الداخلي IP ثابت؟
هل البورت صحيح؟
هل السيرفر يعمل فعلًا على هذا البورت؟
هل Firewall داخل الجهاز يسمح؟
هل الراوتر لديه Public IP حقيقي؟
هل أنت خلف CGNAT؟
```

---

## المشكلة 4: الألعاب تظهر Strict NAT

قد يكون السبب:

```text
CGNAT
Double NAT
Port Forwarding غير مضبوط
UPnP مغلق
Firewall يمنع الاتصالات
```

---

## المشكلة 5: بعض المواقع لا تفتح

قد يكون السبب:

```text
DNS
MTU
Firewall
NAT Timeout
ISP Filtering
```

# أخطاء شائعة

## الخطأ الأول

```text
NAT هو نفسه Firewall
```

غير صحيح.

NAT يترجم العناوين، وFirewall يسمح أو يمنع.

---

## الخطأ الثاني

```text
Private IP يمكن الوصول إليه من الإنترنت مباشرة
```

غير صحيح.

Private IP لا يتم توجيهه على الإنترنت العام.

---

## الخطأ الثالث

```text
Port Forwarding آمن دائمًا
```

غير صحيح.

Port Forwarding يفتح خدمة داخلية للإنترنت، ولازم تأمين الخدمة جيدًا.

---

## الخطأ الرابع

```text
لو عندي NAT إذن أنا آمن بالكامل
```

غير صحيح.

NAT ليس بديلًا عن Firewall وتحديثات الأمان وكلمات المرور القوية.

---

## الخطأ الخامس

```text
كل مشاكل الألعاب سببها الراوتر فقط
```

ليس دائمًا.

قد يكون السبب CGNAT من مزود الخدمة أو Double NAT أو Firewall أو سيرفر اللعبة نفسه.

---

# ملخص سريع

```text
NAT = Network Address Translation
PAT = Port Address Translation
Private IP = داخل الشبكة المحلية
Public IP = على الإنترنت
Static NAT = واحد لواحد
Dynamic NAT = من Pool
PAT / NAT Overload = أجهزة كثيرة على Public IP واحد
NAT Table = يتذكر التحويلات
Port Forwarding = يسمح بدخول اتصال من الخارج لجهاز داخلي
CGNAT = NAT عند مزود الخدمة
Double NAT = وجود NAT مرتين
```

---

# أهم القواعد

```text
Private IP لا يخرج إلى الإنترنت مباشرة
```

```text
NAT يغير Source IP عند الخروج
```

```text
PAT يغير Source IP و Source Port
```

```text
الراوتر يستخدم NAT Table لإرجاع الرد للجهاز الصحيح
```

```text
Port Forwarding مطلوب لو تريد اتصالًا يبدأ من الإنترنت إلى جهاز داخلي
```

```text
NAT ليس Firewall
```

```text
IPv6 يقلل الحاجة إلى NAT لكنه يحتاج Firewall
```
