# IP Addressing and Subnetting

## ما هو عنوان الـ IP؟

عنوان الـ IP هو العنوان المنطقي للجهاز داخل الشبكة.

يعني لو عندك جهاز متصل بالراوتر، لازم يكون له IP حتى يستطيع إرسال واستقبال البيانات.

مثال:

```text
192.168.1.10
```

يمكن تشبيه الـ IP بعنوان البيت.

```text
IP Address = عنوان البيت
MAC Address = هوية كرت الشبكة
Port = رقم الشقة أو الخدمة داخل الجهاز
```

---

## لماذا نحتاج IP Address؟

نحتاج IP حتى تعرف الأجهزة والراوترات:

```text
من أين جاءت البيانات؟
وإلى أين يجب أن تذهب؟
```

مثال:

```text
Source IP      = 192.168.1.10
Destination IP = 142.250.80.46
```

هذا يعني:

```text
الجهاز 192.168.1.10 يريد إرسال بيانات إلى السيرفر 142.250.80.46
```

---

# أنواع IP

## 1. IPv4

IPv4 هو الإصدار الأكثر استخدامًا.

يتكون من 32 bit ويكتب بهذا الشكل:

```text
192.168.1.5
8.8.8.8
142.250.80.46
```

كل رقم بين النقاط يسمى:

```text
Octet
```

مثال:

```text
192 . 168 . 1 . 5
```

كل Octet يمكن أن يكون من:

```text
0 إلى 255
```

لأن كل Octet يتكون من 8 bits.

---

## 2. IPv6

IPv6 هو الإصدار الأحدث.

يتكون من 128 bit ويكتب غالبًا بهذا الشكل:

```text
2001:0db8:85a3:0000:0000:8a2e:0370:7334
```

ظهر IPv6 لأن عدد عناوين IPv4 محدود.

IPv4 يوفر تقريبًا:

```text
4.3 billion addresses
```

وهذا لم يعد كافيًا لكل أجهزة العالم.

---

# Public IP and Private IP

## Public IP

الـ Public IP هو العنوان الذي يظهر على الإنترنت.

يعطيه لك مزود الخدمة ISP.

مثال:

```text
41.33.20.5
```

هذا العنوان يجب أن يكون فريدًا على مستوى الإنترنت.

يعني لا يمكن أن يوجد جهازان على الإنترنت بنفس Public IP في نفس الوقت.

---

## Private IP

الـ Private IP يستخدم داخل الشبكات المحلية فقط.

مثال:

```text
192.168.1.10
```

هذا العنوان لا يظهر مباشرة على الإنترنت.

يعني ممكن يكون عندك في بيتك جهاز IP الخاص به:

```text
192.168.1.10
```

وفي بيت شخص آخر عنده جهاز بنفس العنوان:

```text
192.168.1.10
```

ولا توجد مشكلة، لأن كل واحد داخل شبكة محلية مختلفة.

---

# Private IP Ranges

هذه هي المساحات المخصصة للـ Private IP:

```text
10.0.0.0      - 10.255.255.255
172.16.0.0    - 172.31.255.255
192.168.0.0   - 192.168.255.255
```

أشهر واحدة في شبكات المنازل:

```text
192.168.1.0/24
```

مثال أجهزة داخل البيت:

```text
Router  = 192.168.1.1
Laptop  = 192.168.1.10
Phone   = 192.168.1.11
Printer = 192.168.1.20
```

---

# كيف يخرج Private IP إلى الإنترنت؟

الـ Private IP لا يستطيع الخروج مباشرة إلى الإنترنت.

لذلك يستخدم الراوتر تقنية:

```text
NAT = Network Address Translation
```

مثال:

```text
Before NAT:
192.168.1.10

After NAT:
41.33.20.5
```

يعني الراوتر يغير عنوانك الداخلي إلى Public IP حتى تستطيع الوصول للإنترنت.

---

# ما هو Subnet Mask؟

الـ Subnet Mask يحدد أي جزء من الـ IP يمثل الشبكة، وأي جزء يمثل الجهاز.

مثال:

```text
IP Address  = 192.168.1.10
Subnet Mask = 255.255.255.0
```

هنا:

```text
Network Part = 192.168.1
Host Part    = 10
```

يعني الجهاز موجود داخل شبكة:

```text
192.168.1.0
```

---

## مثال بسيط

لو عندنا:

```text
IP Address  = 192.168.1.10
Subnet Mask = 255.255.255.0
```

فهذا يعني أن الشبكة هي:

```text
192.168.1.0/24
```

والأجهزة داخل نفس الشبكة تكون مثل:

```text
192.168.1.1
192.168.1.2
192.168.1.50
192.168.1.200
```

لكن هذا الجهاز ليس في نفس الشبكة:

```text
192.168.2.10
```

لأن الجزء الثالث مختلف:

```text
192.168.1.x
192.168.2.x
```

---

# CIDR Notation

بدل كتابة:

```text
255.255.255.0
```

يمكن كتابتها بشكل مختصر:

```text
/24
```

مثال:

```text
192.168.1.0/24
```

معناها:

```text
أول 24 bit للشبكة
والباقي للأجهزة
```

---

## أشهر Subnet Masks

| CIDR | Subnet Mask | عدد العناوين تقريبًا |
|---|---|---|
| /8 | 255.0.0.0 | 16,777,216 |
| /16 | 255.255.0.0 | 65,536 |
| /24 | 255.255.255.0 | 256 |
| /25 | 255.255.255.128 | 128 |
| /26 | 255.255.255.192 | 64 |
| /27 | 255.255.255.224 | 32 |
| /28 | 255.255.255.240 | 16 |
| /30 | 255.255.255.252 | 4 |
| /32 | 255.255.255.255 | عنوان واحد فقط |

# Network Address

الـ Network Address هو أول عنوان في الشبكة.

مثال:

```text
192.168.1.0/24
```

هنا عنوان الشبكة هو:

```text
192.168.1.0
```

لا يتم إعطاؤه لجهاز عادي.

لأنه يمثل الشبكة نفسها.

---

# Broadcast Address

الـ Broadcast Address هو آخر عنوان في الشبكة.

مثال:

```text
192.168.1.0/24
```

آخر عنوان هو:

```text
192.168.1.255
```

يستخدم لإرسال رسالة لكل الأجهزة داخل نفس الشبكة.

لا يتم إعطاؤه لجهاز عادي.

---

# Usable Hosts

في أي شبكة IPv4، غالبًا نحذف عنوانين:

```text
Network Address
Broadcast Address
```

مثال:

```text
192.168.1.0/24
```

عدد العناوين الكلي:

```text
256
```

لكن العناوين القابلة للاستخدام:

```text
256 - 2 = 254
```

يعني الأجهزة يمكن أن تأخذ:

```text
192.168.1.1 إلى 192.168.1.254
```

---

# مثال كامل على /24

لو الشبكة:

```text
192.168.1.0/24
```

فالتفاصيل تكون:

```text
Network Address   = 192.168.1.0
First Usable IP   = 192.168.1.1
Last Usable IP    = 192.168.1.254
Broadcast Address = 192.168.1.255
Subnet Mask       = 255.255.255.0
Usable Hosts      = 254
```

---

# Default Gateway

الـ Default Gateway هو الراوتر الذي يخرج جهازك من الشبكة المحلية إلى شبكات أخرى.

مثال:

```text
Device IP       = 192.168.1.10
Default Gateway = 192.168.1.1
```

لو الجهاز يريد إرسال بيانات إلى جهاز داخل نفس الشبكة:

```text
192.168.1.20
```

سيرسل له مباشرة داخل الشبكة المحلية.

لكن لو يريد إرسال بيانات إلى:

```text
8.8.8.8
```

فهذا خارج الشبكة، لذلك يرسلها إلى:

```text
Default Gateway
```

---

# كيف يعرف الجهاز هل الوجهة داخل الشبكة أم خارجها؟

الجهاز يستخدم:

```text
IP Address
Subnet Mask
```

حتى يحسب هل Destination IP داخل نفس الشبكة أم لا.

مثال:

```text
My IP      = 192.168.1.10
Mask       = 255.255.255.0
My Network = 192.168.1.0/24
```

لو الوجهة:

```text
192.168.1.20
```

فهي داخل نفس الشبكة.

لكن لو الوجهة:

```text
142.250.80.46
```

فهي خارج الشبكة.

إذن يرسلها إلى:

```text
Default Gateway
```

---

# مثال عملي: جهاز يفتح google.com

تخيل جهازك بياناته كالتالي:

```text
IP Address      = 192.168.1.10
Subnet Mask     = 255.255.255.0
Default Gateway = 192.168.1.1
DNS Server      = 8.8.8.8
```

ثم كتبت في المتصفح:

```text
google.com
```

الخطوات:

```text
1. الجهاز يسأل DNS عن IP الخاص بـ google.com
2. DNS يرد مثلًا: 142.250.80.46
3. الجهاز يقارن 142.250.80.46 مع شبكته 192.168.1.0/24
4. يكتشف أن Google خارج الشبكة المحلية
5. يرسل البيانات إلى Default Gateway
6. يستخدم ARP لمعرفة MAC Address الخاص بالراوتر
7. الراوتر يستلم Packet ويوجهها إلى الإنترنت
8. NAT يغير Private IP إلى Public IP
```

# مقارنة IP و MAC

| المقارنة | IP Address | MAC Address |
|---|---|---|
| الطبقة | Layer 3 | Layer 2 |
| النوع | Logical Address | Physical Address |
| يتغير؟ | ممكن يتغير حسب الشبكة | ثابت غالبًا في كرت الشبكة |
| يستخدمه | Router | Switch |
| الهدف | تحديد الشبكة والوجهة النهائية | تحديد الجهاز داخل الشبكة المحلية |

---

# مقارنة Public و Private IP

| المقارنة | Public IP | Private IP |
|---|---|---|
| المكان | الإنترنت | الشبكة المحلية |
| يعطيه من؟ | ISP | Router / DHCP |
| فريد عالميًا؟ | نعم | لا |
| مثال | 41.33.20.5 | 192.168.1.10 |
| يحتاج NAT؟ | لا | نعم للخروج للإنترنت |

# أخطاء شائعة

## 1. جهاز أخذ IP يبدأ بـ 169.254

مثال:

```text
169.254.10.20
```

هذا غالبًا معناه أن الجهاز لم يحصل على IP من DHCP.

هذا يسمى:

```text
APIPA
```

ويحدث عندما يفشل الجهاز في الوصول إلى DHCP Server.

---

## 2. IP صحيح لكن لا يوجد إنترنت

قد تكون المشكلة في:

```text
Default Gateway
DNS Server
NAT
Firewall
ISP
```

---

## 3. جهازان بنفس IP

لو جهازان داخل نفس الشبكة أخذا نفس IP، يحدث:

```text
IP Conflict
```

وقد يفشل الاتصال أو يصبح غير مستقر.

---

# ملخص سريع

```text
IP Address = عنوان منطقي للجهاز
Subnet Mask = يحدد الشبكة والجزء الخاص بالأجهزة
Default Gateway = الراوتر الذي يخرجك إلى خارج الشبكة
Public IP = عنوانك على الإنترنت
Private IP = عنوانك داخل الشبكة المحلية
NAT = يحول Private IP إلى Public IP
```

---

# أهم القواعد

```text
إذا كانت الوجهة داخل نفس الشبكة:
الجهاز يرسل مباشرة للوجهة باستخدام ARP لمعرفة MAC الخاص بها
```

```text
إذا كانت الوجهة خارج الشبكة:
الجهاز يرسل إلى Default Gateway باستخدام ARP لمعرفة MAC الراوتر
```

```text
Router يستخدم IP
Switch يستخدم MAC
Port يستخدمه TCP/UDP لتحديد التطبيق
```
