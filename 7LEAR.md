# فهم الـ 7 Layers (نموذج OSI) بشكل عميق

## لماذا وُجد هذا النموذج أصلاً؟

في السبعينيات، كل شركة تصنع أجهزة شبكات بطريقتها الخاصة:
- **IBM** لها نظامها
- **Cisco** لها نظامها
- **Digital** لها نظامها

**المشكلة:** أجهزة IBM لا تتحدث مع أجهزة Cisco!

**الحل:** منظمة **ISO** قالت: "هيا نتفق على معايير موحدة نقسم فيها عملية الاتصال إلى 7 طبقات، وكل شركة تبرمج طبقاتها حسب هذه المعايير."

---

## شرح كل طبقة بالتفصيل

---

### الطبقة 7: التطبيق (Application Layer)

**هي الطبقة الوحيدة التي يراها المستخدم**

```
┌─────────────────────────────┐
│     ما تراه على شاشتك       │
│  المتصفح، الواتساب، البريد  │
└─────────────────────────────┘
```

**ماذا تفعل؟**
- تتواصل مباشرة مع التطبيقات
- تحدد نوع البيانات المطلوبة (صور، نصوص، فيديو)

**البروتوكولات الشهيرة:**
| البروتوكول | الوظيفة |
|-----------|---------|
| HTTP/HTTPS | تصفح الويب |
| FTP | نقل الملفات |
| SMTP | إرسال البريد |
| DNS | ترجمة أسماء المواقع |

**⚠️ ملاحظة مهمة:** المتصفح نفسه ليس هو الطبقة 7، بل المتصفح **يستخدم** بروتوكولات الطبقة 7 (مثل HTTP)

---

### الطبقة 6: العرض (Presentation Layer)

**المترجم والمحرر**

```
┌─────────────────────────────┐
│  قبل: {"name":"ahmed"}      │
│  بعد: ضغط + تشفير           │
└─────────────────────────────┘
```

**ماذا تفعل؟**
1. **الترميز (Encoding):** تحويل البيانات لصيغة مفهومة
   - ASCII للنصوص
   - JPEG للصور
   - MPEG للفيديو

2. **التشفير (Encryption):** حماية البيانات
   - SSL/TLS يعمل هنا

3. **الضغط (Compression):** تقليل حجم البيانات
   - ZIP، GZIP

**مثال واقعي:**
أنت تكتب رسالة بالعربية، الطبقة 6 تحولها لـ Unicode ثم تضغطها ثم تشفرها قبل إرسالها

---

### الطبقة 5: الجلسة (Session Layer)

**مدير الاتصالات**

```
┌─────────────────────────────┐
│  افتح جلسة ← حافظ عليها → أغلقها │
└─────────────────────────────┘
```

**ماذا تفعل؟**
1. **فتح الجلسة:** المصافحة الأولى بين الأجهزة
2. **الحفاظ عليها:** التأكد أن الاتصال لم ينقطع
3. **إغلاق الجلسة:** إنهاء الاتصال بأمان
4. **إعادة الاستئناف:** إذا انقطع الاتصال، تعرف من أين توقفت

**مثال واقعي:**
عندما تدخل موقع البنك وتعمل "Login"، الطبقة 5 تنشئ جلسة وتحافظ عليها حتى تعمل "Logout"

**البروتوكولات:**
- NetBIOS
- RPC (Remote Procedure Call)
- PPTP ( يستخدم في بعض أنواع VPN)

---

### الطبقة 4: النقل (Transport Layer)

**شركة الشحن والمتابعة** ⭐ مهمة جداً

```
┌─────────────────────────────┐
│  البيانات الكبيرة           │
│  ┌────┐ ┌────┐ ┌────┐       │
│  │جزء1│ │جزء2│ │جزء3│       │
│  └────┘ └────┘ └────┘       │
│  + رقم تتبع لكل جزء         │
└─────────────────────────────┘
```

**ماذا تفعل؟**
1. **تقسيم البيانات:** الملف الكبير يُقسم لقطع صغيرة
2. **إضافة أرقام المنافذ (Port Numbers):**
   - البورت 80 = الويب
   - البورت 443 = الويب الآمن
   - البورت 53 = DNS

**البروتوكولان الرئيسيان:**

| الميزة | TCP | UDP |
|--------|-----|-----|
| **السرعة** | أبطأ | أسرع بكثير |
| **الموثوقية** | يتأكد من وصول كل شيء | يرسل ولا ينتظر |
| **الترتيب** | يرتب البيانات | لا يهمني الترتيب |
| **الاستخدام** | التصفح، البريد، الملفات | البث المباشر، الألعاب، الصوت |

**مثال TCP:**
```
أنت: أرسلت 3 أجزاء؟
الخادم: وصلني 1 و 3، لم يصلني 2
أنت: سأعيد إرسال 2
الخادم: تمام، الكامل وصل مرتب ✓
```

**مثال UDP:**
```
أنت: خذ 3 أجزاء
الخادم: (لا يرد، لا يهمه إن فُقد شيء)
```

---

### الطبقة 3: الشبكة (Network Layer)

**نظام التوصيل والملاحة** ⭐ مهمة جداً

```
┌─────────────────────────────┐
│  من: 192.168.1.5            │
│  إلى: 142.250.80.46 (Google)│
│  أفضل طريق: عبر الطريق F    │
└─────────────────────────────┘
```

**ماذا تفعل؟**
1. **إضافة عنوان IP المصدر والوجهة**
2. **التوجيه (Routing):** اختيار أفضل طريق للبيانات
3. **التجزئة (Fragmentation):** إذا كانت الحزمة كبيرة جداً على شبكة معينة

**الجهاز الأساسي:** **الموجه (Router)** يعمل في هذه الطبقة

**البروتوكولات:**
| البروتوكول | الوظيفة |
|-----------|---------|
| IPv4 | العنونة (مثل 192.168.1.1) |
| IPv6 | العنونة الجديدة (أطول) |
| ICMP | اختبار الاتصال (Ping) |
| OSPF | بروتوكول التوجيه |

**مثال واقعي:**
مثل تطبيق **خرائط جوجل**، يخبرك بأفضل طريق لتصل لوجهتك

---

### الطبقة 2: ربط البيانات (Data Link Layer)

**التوصيل داخل المبنى الواحد**

```
┌─────────────────────────────┐
│  من: AA:BB:CC:DD:EE:FF      │
│  إلى: 11:22:33:44:55:66      │
│  (عناوين MAC الفيزيائية)     │
└─────────────────────────────┘
```

**ماذا تفعل؟**
1. **إضافة عناوين MAC:** العنوان الفيزيائي لكرت الشبكة
2. **اكتشاف الأخطاء:** فحص إذا تضررت البيانات في الطريق
3. **التحكم في الوصول:** من يرسل ومتى (لتجنب التصادم)

**الجهاز الأساسي:** **المبدل (Switch)** يعمل في هذه الطبقة

**الفرق بين الطبقة 2 و 3:**
```
الطبقة 3 (IP):   مصر ← السعودية ← الإمارات  (بين دول)
الطبقة 2 (MAC):  داخل المبنى فقط             (داخل غرفة)
```

---

### الطبقة 1: الفيزيائية (Physical Layer)

**الكابلات والإشارات**

```
┌─────────────────────────────┐
│  01010101 ← إشارات كهربائية  │
│  أو ضوء ← ألياف بصرية       │
│  أو موجات ← WiFi            │
└─────────────────────────────┘
```

**ماذا تفعل؟**
1. **تحويل البيانات لإشارات:** Bits → كهرباء/ضوء/موجات
2. **تحديد المواصفات:** نوع الكابل، طول المسموح، نوع الموصل

**الأجهزة:**
| الجهاز | الوظيفة |
|--------|---------|
| كابل Ethernet | نقل كهربائي |
| الألياف البصرية | نقل بالضوء |
| المكرر (Repeater) | تقوية الإشارة |
| Hub | توزيع الإشارة (قديم) |

## وحدات البيانات في كل طبقة (PDU)

كل طبقة تسمي "البيانات" باسم مختلف:

```
الطبقة 7-5:  Data      (بيانات)
الطبقة 4:    Segment   (قطعة)
الطبقة 3:    Packet    (حزمة)
الطبقة 2:    Frame     (إطار)
الطبقة 1:    Bits      (بتات: 0 و 1)
```

---

## التغليف (Encapsulation) بالصورة

عند إرسال صورة:

```
البيانات الأصلية (صورة)
       ↓
═══ طبقة 4: تُضاف Port Numbers  ═══→ Segment
       ↓
═══ طبقة 3: تُضاف عناوين IP     ═══→ Packet
       ↓
═══ طبقة 2: تُضاف عناوين MAC    ═══→ Frame
       ↓
═══ طبقة 1: تتحول لـ 01010101   ═══→ Bits
```

كل طبقة تضع بياناتها في **Header (رأس)** أمام البيانات:

```
┌───────┬───────┬───────┬──────────────┐
│MAC Hdr│IP Hdr │TCP Hdr│   البيانات   │
└───────┴───────┴───────┴──────────────┘
  طبقة2  طبقة3  طبقة4    طبقة7,6,5
```

```
7 التطبيق:    "ماذا يريد المستخدم؟"
6 العرض:      "كيف نُنسّق البيانات؟"
5 الجلسة:     "هل نحن متصلون؟"
4 النقل:      "كيف نوصل البيانات بأمان؟"
3 الشبكة:     "ما هو أفضل طريق؟"
2 ربط البيانات:"من لمن داخل هذه الشبكة؟"
1 الفيزيائية:  "كيف نرسل الإشارات؟"
```
